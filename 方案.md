# 社交分享平台最终版项目方案

## 1. 项目定位

本项目跳过 Web 三件套原型、Vue3 入门拆分和单独 JavaWeb 学习阶段，直接进入最终阶段：构建一个前后端分离的社交分享平台。

最终版平台采用 `Vue3 + JavaWeb/Spring Boot + MySQL` 技术路线，实现用户注册登录、文字分享、图片上传、信息流浏览、评论、点赞、个人主页、权限控制和管理员内容管理。前端负责页面交互和用户体验，后端负责认证、数据持久化、文件上传、权限校验和 API 服务。

## 2. 技术架构

### 2.1 前端技术栈

- Vue3：构建组件化前端应用。
- Vite：开发环境和构建工具。
- Vue Router：页面路由管理。
- Pinia 或 Composition API Store：用户、内容、评论、点赞状态管理。
- Axios：统一封装 API 请求。
- CSS：可使用原生 CSS，也可后续升级为 Tailwind CSS。

### 2.2 后端技术栈

- Java 17 或 Java 21。
- Spring Boot：后端主框架。
- Spring MVC：RESTful API。
- MyBatis 或 MyBatis-Plus：数据库访问。
- MySQL：业务数据持久化。
- Spring Security 或自定义拦截器：登录认证与权限控制。
- MultipartFile：图片上传处理。

### 2.3 数据流

用户在 Vue 页面操作后，前端通过 Axios 请求后端 API。后端完成参数校验、身份识别、权限判断、业务处理和数据库读写，再返回统一 JSON 响应。图片上传由前端提交 `multipart/form-data`，后端保存文件并返回图片访问地址。

## 3. 核心功能

### 3.1 用户认证

支持用户注册、登录、退出和登录状态保持。

注册字段建议：

- 用户名。
- 昵称。
- 密码。
- 确认密码。

登录成功后，后端返回 token 或 session 状态，前端保存登录信息并在请求头中携带认证凭证。

### 3.2 角色与权限

系统包含三类角色：

- 游客：可以浏览公开内容，不能发布、评论、点赞。
- 普通用户：可以发布文字和图片、评论、点赞、删除自己的内容。
- 管理员：可以管理全部内容、删除违规分享或评论、隐藏或恢复内容。

权限必须由后端最终校验。前端可以隐藏按钮和入口，但不能作为安全依据。

### 3.3 信息流浏览

首页展示公开分享内容，按发布时间倒序排列。

信息流卡片包含：

- 作者头像和昵称。
- 发布时间。
- 分享标题。
- 分享正文。
- 分享图片。
- 点赞数。
- 评论数。
- 操作按钮。

支持搜索和筛选：

- 按关键词搜索标题、正文、作者。
- 只看图片内容。
- 热门内容。
- 我的发布。
- 管理员查看隐藏内容。

### 3.4 文字发布

登录用户可以发布文字内容。

校验规则：

- 标题不能为空，建议 2-40 个字符。
- 正文不能为空，建议 1-500 个字符。
- 内容需要做 XSS 防护，前端展示时避免直接插入 HTML。

### 3.5 图片上传

发布内容时支持上传一张图片。

要求：

- 允许 `jpg`、`jpeg`、`png`、`webp`。
- 单张图片建议不超过 2MB 或 5MB。
- 前端上传前预览。
- 后端校验文件类型和大小。
- 后端返回图片访问 URL，数据库保存 URL。

### 3.6 评论功能

登录用户可以对分享发表评论。

评论功能包括：

- 评论列表。
- 评论发布。
- 评论删除。
- 评论数量统计。
- 普通用户删除自己的评论。
- 管理员删除任意评论。

评论建议限制 1-200 个字符。

### 3.7 点赞功能

登录用户可以点赞或取消点赞。

规则：

- 同一用户对同一条分享只能点赞一次。
- 再次点击取消点赞。
- 后端使用点赞关系表保证唯一性。
- 前端根据当前用户点赞状态显示不同按钮状态。

### 3.8 个人主页

个人主页展示：

- 用户昵称、头像、简介。
- 用户发布的分享。
- 发布数量、点赞数量、评论数量。
- 当前登录用户访问自己主页时，可以编辑个人信息。

### 3.9 管理员功能

管理员后台或管理视图包含：

- 内容列表。
- 删除任意分享。
- 隐藏或恢复分享。
- 删除任意评论。
- 查看平台统计。

平台统计建议包括：

- 用户数。
- 分享数。
- 评论数。
- 点赞数。
- 隐藏内容数。

## 5. 视觉与交互方向

界面应像真实社区产品，而不是宣传页。

设计原则：

- 信息流是首页核心。
- 发布入口清晰。
- 评论和点赞交互轻量直接。
- 管理员功能安静、清晰、可控。
- 卡片圆角不超过 8px。
- 所有按钮有 hover、active、disabled 和 focus-visible 状态。
- 移动端优先保证阅读和发布体验。
- 不使用 emoji 作为功能图标，后续可使用 Lucide 图标。

推荐视觉：

- 主色：紫色系，用于品牌、导航和焦点状态。
- 行动色：绿色，用于发布、保存、成功提示。
- 危险色：红色，用于删除、隐藏、错误提示。
- 背景：近白色或浅灰紫，保证内容可读。
- 字体：系统无衬线字体优先，标题略粗，正文保持舒适行高。

## 6. 数据库设计

### 6.1 users 用户表

```sql
CREATE TABLE users (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  username VARCHAR(50) NOT NULL UNIQUE,
  nickname VARCHAR(50) NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  avatar_url VARCHAR(255),
  bio VARCHAR(255),
  role VARCHAR(20) NOT NULL DEFAULT 'USER',
  status VARCHAR(20) NOT NULL DEFAULT 'ACTIVE',
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL
);
```

### 6.2 posts 分享表

```sql
CREATE TABLE posts (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  title VARCHAR(80) NOT NULL,
  content TEXT NOT NULL,
  image_url VARCHAR(255),
  visibility VARCHAR(20) NOT NULL DEFAULT 'PUBLIC',
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 6.3 comments 评论表

```sql
CREATE TABLE comments (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  post_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  content VARCHAR(300) NOT NULL,
  created_at DATETIME NOT NULL,
  updated_at DATETIME NOT NULL,
  FOREIGN KEY (post_id) REFERENCES posts(id),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 6.4 likes 点赞表

```sql
CREATE TABLE likes (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  post_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  created_at DATETIME NOT NULL,
  UNIQUE KEY uk_post_user (post_id, user_id),
  FOREIGN KEY (post_id) REFERENCES posts(id),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 6.5 files 文件表

```sql
CREATE TABLE files (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL,
  original_name VARCHAR(255) NOT NULL,
  storage_path VARCHAR(255) NOT NULL,
  url VARCHAR(255) NOT NULL,
  content_type VARCHAR(80) NOT NULL,
  size BIGINT NOT NULL,
  created_at DATETIME NOT NULL,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

## 7. API 设计

统一响应格式：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

认证接口：

```text
POST /api/auth/register
POST /api/auth/login
POST /api/auth/logout
GET  /api/auth/me
```

用户接口：

```text
GET  /api/users/{id}
PUT  /api/users/me
```

分享接口：

```text
GET    /api/posts
POST   /api/posts
GET    /api/posts/{id}
PUT    /api/posts/{id}
DELETE /api/posts/{id}
PATCH  /api/posts/{id}/visibility
```

评论接口：

```text
GET    /api/posts/{id}/comments
POST   /api/posts/{id}/comments
DELETE /api/comments/{id}
```

点赞接口：

```text
POST   /api/posts/{id}/like
DELETE /api/posts/{id}/like
```

文件接口：

```text
POST /api/files/upload
```

管理接口：

```text
GET /api/admin/stats
GET /api/admin/posts
```

## 9. 开发顺序

虽然跳过前三个学习阶段，但最终阶段仍建议按功能闭环推进：

1. 搭建后端项目、数据库连接和统一响应结构。
2. 创建数据库表和实体模型。
3. 实现注册、登录、当前用户接口。
4. 搭建 Vue3 项目、路由和 Axios 封装。
5. 实现登录、注册和登录状态保持。
6. 实现分享列表、发布、详情、删除接口和页面。
7. 实现图片上传并接入发布流程。
8. 实现评论列表、发布、删除。
9. 实现点赞和取消点赞。
10. 实现个人主页。
11. 实现管理员管理页和权限控制。
12. 完成错误处理、加载状态、响应式适配和最终测试。

## 10. 安全要求

必须实现：

- 密码不能明文存储，必须保存哈希。
- 后端必须校验登录状态。
- 后端必须校验用户是否有权限操作目标资源。
- 后端必须校验上传文件类型和大小。
- 前端展示用户输入时避免 XSS。
- API 返回不能暴露密码哈希等敏感字段。
- 管理员接口必须限制管理员角色访问。

## 11. 验收标准

最终项目应满足：

- 用户可以注册、登录、退出。
- 游客可以浏览公开分享。
- 登录用户可以发布文字和图片。
- 登录用户可以评论、点赞、取消点赞。
- 用户可以删除自己的分享和评论。
- 管理员可以删除或隐藏任意分享和评论。
- 首页信息流、发布页、详情页、个人主页、管理页均可正常使用。
- 图片上传成功后刷新页面仍可显示。
- 数据保存到 MySQL。
- 前端所有 API 请求都有加载、成功和失败反馈。
- 移动端 375px 宽度下无横向滚动。
- 权限校验以后端结果为准。
